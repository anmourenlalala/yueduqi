================================================================================
                        上下文API测试服务器使用说明
================================================================================

一、文件说明
================================================================================

1. 测试.bat - 启动测试服务器的批处理脚本
2. test_server.js - 测试服务器代码（Node.js）
3. README.txt - 本说明文件


二、使用方法
================================================================================

1. 确保主服务器正在运行
   - 主服务器应该运行在端口 2333
   - 确保前端应用已打开并加载了文件

2. 启动测试服务器
   - 双击运行 "测试.bat"
   - 或者命令行运行: node test_server.js

3. 测试服务器将在端口 5555 启动
   - 测试端点: http://localhost:5555/test
   - 信息端点: http://localhost:5555/test-info
   - 测试页面: http://localhost:5555/


三、测试端点说明
================================================================================

1. GET http://localhost:5555/test
   - 功能: 测试获取上下文API
   - 说明: 调用主服务器（2333端口）的上下文API
   - 返回: JSON格式的测试结果

2. GET http://localhost:5555/test-info
   - 功能: 测试获取上下文信息API
   - 说明: 调用主服务器（2333端口）的上下文信息API
   - 返回: JSON格式的上下文元信息

3. GET http://localhost:5555/
   - 功能: 测试页面
   - 说明: 在浏览器中打开，提供可视化的测试界面
   - 使用: 点击按钮进行测试


四、测试方法
================================================================================

方法1: 使用浏览器
   1. 启动测试服务器（运行 测试.bat）
   2. 在浏览器中打开 http://localhost:5555/
   3. 点击"测试获取上下文"或"测试上下文信息"按钮
   4. 查看测试结果

方法2: 使用curl
   curl http://localhost:5555/test
   curl http://localhost:5555/test-info

方法3: 使用Python
   ```python
   import requests
   response = requests.get('http://localhost:5555/test')
   print(response.json())
   ```

方法4: 使用JavaScript
   ```javascript
   fetch('http://localhost:5555/test')
       .then(res => res.json())
       .then(data => console.log(data));
   ```


五、测试结果说明
================================================================================

成功响应示例:
{
  "success": true,
  "message": "测试成功",
  "contextLength": 8234,
  "contextPreview": "当前项目显示的文件内容：\n共有 2 个视图：...",
  "timestamp": "2024-01-15T14:30:25.123Z"
}

失败响应示例:
{
  "success": false,
  "message": "获取上下文失败",
  "statusCode": 404,
  "error": {
    "error": "Context not available",
    "message": "前端尚未发送上下文信息，请确保前端应用正在运行并已加载文件"
  },
  "timestamp": "2024-01-15T14:30:25.123Z"
}


六、注意事项
================================================================================

1. 确保主服务器（2333端口）正在运行
2. 确保前端应用已打开并加载了文件
3. 如果测试失败，检查主服务器日志
4. 测试服务器日志会显示详细的调用信息


七、故障排除
================================================================================

问题1: 无法连接到2333端口
解决: 确保主服务器正在运行（运行 server.js）

问题2: 上下文不可用
解决: 确保前端应用已打开并加载了文件

问题3: 端口5555已被占用
解决: 修改 test_server.js 中的 port 变量为其他端口
