【不需要保存的目录和文件】

× node_modules/          (可通过 npm install 重新安装)
× 网页版deepseek/        (子项目，单独管理)
× 完整源码/              (备份文件夹)
× *.zip                  (压缩包备份)
× *.7z                   (压缩包备份)
× package-lock.json      (可选，可通过 npm install 重新生成)

【注意事项】

1. 如果使用Git版本控制，建议创建.gitignore文件，包含：
   node_modules/
   deepseek-profile/
   *.zip
   *.7z
   package-lock.json (可选)

2. 首次在新环境部署时，需要运行：
   npm install

3. deepseek-profile目录会在首次使用DeepSeek功能时自动创建
   - 首次使用需要手动登录DeepSeek（浏览器会自动打开）
   - 登录状态会保存到deepseek-login-state.json
   - 下次启动会自动使用已保存的登录状态

4. 数据文件（prompts/、themes/、workflows/、events/、layouts/）包含用户的配置和数据，务必保存

5. 日志文件（log/、fankui_log/）包含执行历史和反馈数据，可根据需要保存

6. 回收站文件（trash.json）包含删除文件信息，可根据需要保存

7. 内网穿透相关文件（cloudflared.exe）用于公网访问，可根据需要保存
